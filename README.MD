# Test Task Karma 8

## Backend

У меня не получилось использовать ссылки с гугл диска для видео напрямую, поэтому я сделал прокси, который скачивает видео оттуда и отдаёт ссылки на фронт.
Видео скачиваются при первом запуске приложения, так что загрузка немного долгая. Можно было сделать лучше, но эта часть не относится к заданию. Будем считать что у нас есть нормальный бек, который отдаёт ссылки на статику.

## Frontend

- `VideoItem` - создаёт видео и управляет воспрозведением при скролле, так же добавил остановку загрузки если далеко уехала нода.
- `Fetcher` - класс для загрузки ссылок на видео, максимально простой, можно развить добавить пагинацию, какое-то состояние и тд
- `FeedController` - контроллер ленты видео. Дёргает фетчер, создаёт ноды с видео, добавляет видео в DOM
- `SwipeController` - отслеживание событий для перелистываения видео. Вызывает на каждый свайп вниз `FeedController.checkEnd()`, который проверяет что пора загрузить ещё ссылок на видео и добавить.

## Запуск приложения

из корня проекта выполнить:

`npm run reels`

## Запуск в докере

выполнить:

`docker-compose up`

дождаться сборки и открыть в браузере:

`http://127.0.0.1:5500/index.html`
